Session Manager

The dockable Session Manager lets you quickly connect to defined sessions (or to create and connect to new sessions) in the SecureCRT window. The Session Manager pane can be positioned on the left, right, top, or bottom of the SecureCRT window or undocked as a modeless dialog. To change the location, make sure the Session Manager is pinned and then drag it by the toolbar to the desired location.

Note: To use the Connect dialog instead of the Session Manager, uncheck the Use dockable Session Manager option in the General category of the Global Options dialog.

From the Session Manager, you can connect to multiple sessions and folders. You can also select and edit multiple sessions from the Session Manager; only the changed fields will be applied to the selected sessions, allowing specific settings to be changed for a group of sessions. When editing multiple sessions, SecureCRT will display the first session that was selected.

The toolbar on the Session Manager provides quick access to the following actions:

Button

Description

Connect: Causes SecureCRT to connect to the remote server named in the selected session.

Connect in Tab/Tile: Connects the selected session in a tab or tile in the current window.

New Session: Opens the Session Options dialog for a new session.

Cut: Copies the selected session or folder to the clipboard. When it is pasted, the session or folder is moved to the new location.

Copy: Copies the selected session or folder to the clipboard.

Paste: Pastes a session or folder that was previously cut or copied into the specified folder.

Delete: Deletes the selected sessions and/or folders.

Properties: Opens the Session Options dialog for the selected session.

Find: Searches the contents of the session tree for a given string. The accelerator for this command is CTRL+F, and pressing the COMMAND+G key will result in a "find next" operation.

New Folder: Creates a new folder in the folder tree list.

Create Desktop Shortcut: Creates a desktop shortcut to the selected session.
